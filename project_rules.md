# 前端项目代码与资源规范（Trae 专用）

## 1. 代码规范
- 使用 ESLint + Prettier：2空格缩进、单引号、语句末尾加分号。
- 变量/函数：小驼峰（camelCase）；常量：全大写下划线（SNAKE_CASE）；类/组件：大驼峰（PascalCase）。
- 禁止重复代码：公共逻辑抽成函数/hook/utils。
- 删除无用代码、注释、console.log、debugger。
- 所有文件末尾留一空行。

## 2. 资源文件规范（图片/字体）
### 2.1 图片
- 存放目录：
  - 页面模块图：`assets/images/pages/[模块名]/`
  - 公共组件图：`assets/images/components/`
  - 图标：`assets/images/icons/`
- 命名规范：`[用途]_[描述]_[尺寸/状态].[后缀]`
  - 示例：`btn_submit_normal.png`、`banner_home_1920x400.jpg`、`icon_search.svg`
- 格式：优先 WebP/SVG，次选 PNG；禁止重复图片（md5 相同只留一个）。

### 2.2 字体
- 目录：`assets/fonts/[字体名]/`
- 命名：`[字体名]_[字重].[后缀]`，如 `inter_400.woff2`
- 只保留项目实际用到的字体文件。

## 3. CSS 样式优化
- 使用统一变量：颜色、间距、圆角、字体大小。
- 合并重复样式，提取公共类。
- 删除未使用的选择器、属性。
- 选择器扁平化，避免深层嵌套（最多 2 层）。
- 优先使用 Flex/Grid，减少浮动。
- 单位：rem/vw 为主，px 仅用于边框/小尺寸。

## 4. 交付要求
- 执行完后输出：
  1) 代码规范问题清单及修复
  2) 重复代码清单及抽离方案
  3) 重复图片/字体清单及删除建议
  4) CSS 优化前后对比
  5) 资源文件最终目录结构
